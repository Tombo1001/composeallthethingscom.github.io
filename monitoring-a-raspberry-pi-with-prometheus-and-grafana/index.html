    Deploying a Prometheus and Grafana Monitoring Setup with Docker Compose#

    If you’re looking to monitor your Raspberry Pi’s system performance, Prometheus and Grafana are a powerful duo that can help. And using Docker Compose makes setting up this monitoring system a breeze. In this blog post, we’ll walk through how to set up this monitoring system step-by-step using Docker Compose.

    The Components#

    There are three main components to this monitoring setup:

    1. Raspberry Pi running the Node Exporter
    2. Central Prometheus listener, which serves as the data source for Grafana
    3. Grafana Dashboard

    Setting Up the Raspberry Pi with Node Exporter#

    First, we need to set up the Raspberry Pi with the Node Exporter. The Node Exporter is a Prometheus exporter that collects system metrics like CPU usage, memory usage, and disk I/O statistics.

    One way to do this is to install Node Exporter directly on the Pi. Another way is to use a Docker container, which allows for easier management and distribution. In this example, we’ll be using the Docker container approach.

    Here’s an example docker-compose.yml file to set up the Node Exporter on the Pi:

    version: '3'

services:
  node-exporter:
    image: prom/node-exporter
    ports:
      - 9100:9100
    networks:
      - monitoring

networks:
  monitoring:

    This creates a node-exporter service running the Prometheus Node Exporter image, with the container’s port 9100 mapped to the host’s port 9100.

    Creating the Prometheus and Grafana Services#

    Next, we need to set up the Prometheus and Grafana services using Docker Compose.

    version: '3'

services:
  node-exporter:
    image: prom/node-exporter
    ports:
      - 9100:9100
    networks:
      - monitoring

  prometheus:
    image: prom/prometheus
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    ports:
      - 9090:9090
    depends_on:
      - node-exporter
    networks:
      - monitoring

  grafana:
    image: grafana/grafana
    ports:
      - 3000:3000
    depends_on:
      - prometheus
    networks:
      - monitoring

networks:
  monitoring:

    This creates three services:

    1. The node-exporter service running the Prometheus Node Exporter as we saw earlier.
    2. The prometheus service running the Prometheus Server. The prometheus.yml configuration file needs to be created manually. Here’s an example configuration file:
    global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'node-exporter'
    static_configs:
      - targets: ['node-exporter:9100']

    This configuration instructs Prometheus to scrape the metrics exposed by the node-exporter service running on the node-exporter service name.

    1. The grafana service running Grafana.

    Setting Up the Grafana Dashboard#

    Finally, we need to set up the Grafana Dashboard to display the Pi’s CPU usage, disk usage, and network usage as a time series graph.

    Here’s an example dashboard configuration that you can import into Grafana:

    apiVersion: 1

providers:
- name: 'Prometheus'
  orgId: 1
  folder: ''
  type: 'prometheus'
  access: 'proxy'
  url: 'http://prometheus:9090'
  basicAuth: false
  isDefault: true

dashboard:
  id: null
  title: 'Raspberry Pi Monitoring'
  hideControls: false
  editable: true
  graphTooltip: 0
  rows:
  - title: 'CPU Usage'
    height: '250px'
    panels:
    - title: 'CPU Percentage'
      type: 'graph'
      span: 12
      fill: 1
      linewidth: 2
      targets:
      - expr: '100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[1m])) * 100)'
  - title: 'Disk Usage'
    height: '250px'
    panels:
    - title: '/mnt/data1'
      type: 'graph'
      span: 12
      fill: 1
      linewidth: 2
      targets:
      - expr: 'node_filesystem_size{mountpoint="/mnt/data1"} - node_filesystem_avail{mountpoint="/mnt/data1"}'
  - title: 'Network Usage'
    height: '250px'
    panels:
    - title: 'Receive Bytes per Second'
      type: 'graph'
      span: 6
      fill: 1
      linewidth: 2
      targets:
      - expr: 'rate(node_network_receive_bytes_total{device!~"lo"}[1m])'
    - title: 'Transmit Bytes per Second'
      type: 'graph'
      span: 6
      fill: 1
      linewidth: 2
      targets:
      - expr: 'rate(node_network_transmit_bytes_total{device!~"lo"}[1m])'
  schemaVersion: 16
  version: 0
  time: null
  timezone: ''

    This dashboard has three panels displaying CPU usage, disk usage, and network usage graphs.

    Conclusion#

    That’s it! You now have a Prometheus and Grafana monitoring setup to monitor your Raspberry Pi’s system performance. Using Docker Compose makes deploying this system quick and easy.
